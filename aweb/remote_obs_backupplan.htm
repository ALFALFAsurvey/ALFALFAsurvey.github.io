ALFALFA Remote Observing Checklist: Current as of February 12, 2006


Things needed before observing

  • laptop (for comfortable viewing of the AO status display) or other workstation....
  • observing checklist
  • this checklist
  • coffee or diet pepsi, prosciutto sandwich, wings, junk food... your choice.

30 minutes before observing

  • Call the telescope operator at (+1) 787 878 2612, Ext. 211, and ask for the password for user "dtusr" at "Observer2". Ask also for the status of ALFA (cover off? amplifiers on? all beams working? anything else new?...)

  • Set up your laptop (or another workstation), login to AO using the a2010 account, and start the AO status display:
    ssh -X a2010@remote.naic.edu
    (the "X" variable sets up all the environment variables relevant for the transmission of graphics accross the link).
    aostatus
    and the pointing monitor
    monpnt

    Note: See Phil's web page for how to access other telescope monitoring windows.

  • NEW! Keep an eye on the aostatus window, and have the a2010.cat file open for editing. As soon as the previous observer is out, check the LST, and estimate and add the slew time (usually around 5 minutes). Put this in for the RA in the galfasrc entry and save the file.

  • On your workstation you will need to use 2 desktops, which will be very creatively referred to as Desk1 and Desk2.

    On Desk1 open a terminal, login to AO using the a2010 account, and start rxview to monitor the status of the IF/LO system:
    ssh -X a2010@remote.naic.edu
    rxview

    On Desk2 open a terminal, login to AO using the a2010 account, and ssh into observer2 using the dtusr account (and the password given to you by the telescope operator):
    ssh -X a2010@remote.naic.edu
    ssh -X dtusr@observer2
    You will use this terminal to run CIMA.

Ready to go!

  • Call the telescope operator and wait for the ok. Only one session of CIMA can run on observer2 at a given time. Tell the operator the approximate AZ,ZA you want so that he/she can start moving the telescope as soon as the previous program is done. Also ask him/her to set up GALSPECT.
  • Start CIMA on observer2 - Invoke the version we are currently using Evolved version 2.1. Type the following 
    >  cima --X
    and then choose the new "Normal Version 2.1". After typing in a2010 as both the project and observer name, you will be authorized by the operator.
  • Follow the instructions in the A2010 observing checklist.
  • If you have problems (e.g. CIMA does not start, you can't select the receiver, the telescope doesn't move...) contact the telescope operator.
  • On Desk1 open another terminal and run Mikael's software for data monitoring. You have to do this on the dataview machine:
    ssh -X a2010@remote.naic.edu
    ssh -X a2010@dataview
    /home/cima/bin/alfaobswin & - shows the power in dB for each beam
    /home/cima/bin/alfamotorwin & - shows ALFA rotation angle
    /home/cima/bin/alfabeamwin & - shows ALFA beam positions
    The waterfall and spectral plots could be seen by typing /home/cima/bin/alfadatawin &, but unfortunately, that slows things down too much. Do NOT open it.
  • On another terminal, likewise login and start up an IDL session on one of the fast linux machines (e.g. fusion01) so that you can run "quicklook". Note these instructions make use of the aliases in the a2010 account.
    ssh -X a2010@remote.naic.edu
    ssh -X a2010@fusion01
    cd /share/a2010/idl/eggidl
    idl
    @wasinit2
    quicklook 'filename'
    where 'filename' is the complete name of the last complete FITS file, e.g. '/share/pserverf.sdb1/wappdata/wapp.20050706.a2010.0003.fits'. You have the option to display the bandpasses for each beam, both pols of the first record of the file. Not as nice as Mikael's data monitoring, but quick and convenient for remote observers.

While observing

  • Keep an eye on the remaining time until the end of the scan on the AO status display. Occasionally it might stop for no reason, meaning that the WAPPs are no longer taking data, and you will receive no error or warning messages from CIMA.
  • Watch the CIMA "AO Observer Display" window, where error and warning messages are displayed. When you see messages concerning the WAPPs (e.g., WAPP(N) DONE adjusting power, WAPP(N) DONE with spectral line observation, etc.), make sure that all four WAPPs are responding. Do not count on CIMA letting you know that one of the WAPPs has stopped working...
  • Check periodically that the declination setting (J2000) is correct. For this, look at the monpnt window to be sure that the line starting with Rslt: shows the correct DecJ. Also check the appearance of the array and its coordinates in the alfabeampos widget. If anything looks amiss, discuss with the telescope operator. In particular, be sure that he has given you control of the telescope!
  • Restart the WAPPS (1) before TOGS at the beginning of the night; (2) at A2010 startup; and (3) before TOGS at the end of the night.
